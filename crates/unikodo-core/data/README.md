# Vendored symbol data

`unikodo-core` builds its symbol database from the sources below. The
`unicode-math` table is vendored verbatim; the `latex`/`lean`/`rocq` tables are
**normalized extracts** (`name<TAB>codepoint(hex)[ ...]<TAB>description`)
regenerated by the `scripts/update-*-symbols.py` scripts; the Typst (`typst`)
scheme is pulled from the [`codex`](https://crates.io/crates/codex) crate at build
time (not vendored here).

| File | Scheme | Upstream | Licence |
| --- | --- | --- | --- |
| `unicode-math-table.tex` | `unicode-math` | [unicode-math](https://github.com/wspr/unicode-math) (Will Robertson et al.) | LPPL v1.3c+ |
| `latex-symbols.tsv` | `latex` | [W3C "XML Entity Definitions for Characters" (`unicode.xml`)](https://www.w3.org/Math/characters/) (David Carlisle et al.) | W3C Software and Document Licence |
| `lean-symbols.tsv` | `lean` | [`leanprover/vscode-lean4`](https://github.com/leanprover/vscode-lean4) `abbreviations.json` | Apache-2.0 |
| `rocq-symbols.tsv` | `rocq` | [`rocq-prover/rocq`](https://github.com/rocq-prover/rocq) `ide/rocqide/default_bindings_src.ml` | LGPL-2.1-only |

Each vendored/derived file is redistributed under **its own** upstream licence
(noted above and in the file's header comment), **not** unikodo's MIT licence.
The derived `.tsv` tables are mechanical extracts of the upstream data, credited
to their projects.

## Updating

From the repository root:

```sh
scripts/update-unicode-math-table.sh    # unicode-math (raw .tex)
scripts/update-latex-symbols.py         # latex   (from W3C unicode.xml)
scripts/update-lean-symbols.py          # lean    (from vscode-lean4)
scripts/update-rocq-symbols.py          # rocq    (from rocq-prover/rocq)
cargo test -p unikodo-core              # validate the parse + counts
```

For Typst, bump the `codex` dependency version in `Cargo.toml`. The
`unicode_math_snapshot_count` test guards the `unicode-math` table against
accidental corruption; adjust it if a deliberate update changes the count.
